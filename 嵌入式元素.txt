Drop-in Element  插入元素
The Drop-in Element lets you embed multiple payment methods with a single integration while offering options to customize the look and feel of the checkout experience.
插件功能允许您通过一次集成嵌入多种支付方式，同时提供自定义结账体验外观和风格的选项。
Create Drop-in Element  创建插入式元素
createElement('dropIn', options?)
Use this function to create an instance of an individual Element.
使用此函数创建单个元素的实例。

Parameters  参数
type
required  必需的
'dropIn'
The type of element you are creating.

options
optional  选修的
DropInElementOptions
Options for creating dropIn Element.

Returns  返回
DropInElement | null
linkTypeScript
import { createElement } from '@airwallex/components-sdk';
const element = await createElement('dropIn', {
  intent_id: 'replace-with-your-intent-id',
  client_secret: 'replace-with-your-client-secret',
  currency: 'replace-with-your-intent-currency',
});
DropInElementOptions
Apply to dropIn element type integration, interface used when createElement called with type dropIn
适用于 dropIn 元素类型集成，当使用 dropIn 类型调用 createElement 时使用的接口

client_secret  客户端密钥
required  必需的
string  细绳
The client_secret of the Payment Intent when Payment Intent is provided. Otherwise, this should be the client_secret of the Customer object.
当提供了支付意图时，此值为支付意图的 client_secret 。否则，此值为 Customer 对象的 client_secret 。

currency  货币
required  必需的
string  细绳
The three-letter ISO currency code representing the currency of the Payment Intent or Payment Consent. Only payment methods supported for the specified currency will be shown on the payment page.
代表付款意向或付款授权所用货币的三字母 ISO 货币代码。付款页面上只会显示支持该指定货币的付款方式。

allowedCardNetworks  允许的卡网络
optional  选修的
CardNetwork[]
CardNetwork []
One or more card networks that you support.
您支持的一个或多个卡网络。

appearance  外貌
optional  选修的
DropInAppearance
Customize the visual appearance of the DropIn Element to match your website's design.
自定义 DropIn 元素的视觉外观，使其与您的网站设计相匹配。

You can configure:  您可以配置：

mode: Choose between 'dark' and 'light' mode.
模式：选择 “深色” 模式或 “浅色” 模式。

variables: Set colorBrand, colorText, colorBackground properties.
变量：设置 colorBrand 、 colorText 、 colorBackground 属性。

rules: Customize the styles of the specific elements.
规则：自定义特定元素的样式。

authorizationType  授权类型
optional  选修的
AuthorizationType  授权类型
The authorization type for the card payment. Set it to pre_auth if you want to place a hold on your customer's card for more than 7 days, i.e., extend the authorization time window. Currently pre_auth is only available when the card brand is Visa or Mastercard. autoCapture will be automatically set to false if you enable pre_auth. Default value is 'final_auth'.
信用卡支付的授权类型。如果您希望冻结客户信用卡超过 7 天（即延长授权时间），请将其设置为 pre_auth 。目前， pre_auth 仅适用于 Visa 或 Mastercard 卡。启用 pre_auth 后， autoCapture 将自动设置为 false 。默认值为 final_auth 。

autoCapture  自动捕获
optional  选修的
boolean  布尔值
Whether the amount should be captured automatically upon successful payment authorization. Set it to false if you want to place a hold on the payment method and capture the funds sometime later. Default value is true.
是否在付款授权成功后自动扣款。如果您希望暂时冻结付款方式并在稍后扣款，请将其设置为 false。 默认值为 true 。

autoSaveCardForFuturePayments
自动保存信用卡信息以备将来支付
optional  选修的
boolean  布尔值
Specifies whether the card payment method should be automatically saved for future transactions.
指定是否应自动保存银行卡支付方式以供将来交易使用。

This parameter is only effective in payment mode when a customer_id is provided.
此参数仅在提供 customer_id 的支付模式下有效。

If set to true, the "Save my card for future payments" checkbox will be preselected by default.
如果设置为 true ，则“保存我的银行卡以便将来付款”复选框将默认预先选中。
-If set to false, the checkbox will remain unchecked, requiring the shopper to manually opt in.
-如果设置为 false ，则复选框将保持未选中状态，需要购物者手动选择加入。

Default value is true
默认值为真

quote_id  报价单
optional  选修的
null | string  空 | 字符串
This field is used for Multi-Currency Pricing(MCP). The merchant can create a quote for the shopper, and pass the quote_id to Drop-in Element. If the quote_id is provided, Drop-in Element will use that quote to convert the amount and currency. And each time the shopper changes the currency, the merchant needs to create a new quote or reuse existing quotes and update the quote_id. If the quote_id is not provided, Drop-in Element will use ACC logic to automatically convert the amount and currency.
此字段用于多币种定价 (MCP)。商家可以为顾客创建报价单，并将 quote_id 传递给 Drop-in Element。如果提供了 quote_id，Drop-in Element 将使用该报价单转换金额和货币。每次顾客更改货币时，商家都需要创建新报价单或重用现有报价单并更新 quote_id。如果未提供 quote_id，Drop-in Element 将使用 ACC 逻辑自动转换金额和货币。

// 1. Call with the container DOM id
element.mount('container-dom-id');

// 2. Find the created DOM in the existing HTML and call with the container DOM element

element.mount(containerElement);
